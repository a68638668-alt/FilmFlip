FilmFlip v2.0
필름 스캔 파일 관리 도구

버전: 2.0.0
문서 기준일: 2026-07-15

===============================================================================
1. 프로그램 소개
===============================================================================

FilmFlip은 필름 스캔 이미지의 순서를 확인하고, 파일명과 폴더명을 촬영 정보에
맞게 일괄 정리하며, JPEG 사진의 실제 EXIF 정보까지 편집할 수 있는 데스크톱
프로그램입니다.

v2.0은 v1.2의 파일 정리 기능을 유지하면서 메인 화면과 각 편집 창을 전면
개편하고, EXIF 편집·썸네일 크기·테마·롤 기준 컷수 설정을 추가했습니다.


===============================================================================
2. 주요 기능
===============================================================================

- 이미지 폴더 선택 및 폴더 드래그 앤 드롭
- 필름 프레임 형태의 썸네일 목록
- 선택 사진 미리보기 및 해상도/파일 형식 확인
- 이미지 순서 변경 및 역순 번호 적용
- 촬영 정보 조합을 이용한 파일명 일괄 변경
- 촬영 정보 조합을 이용한 폴더명 변경
- 촬영 프리셋 저장·수정·삭제·순서 관리
- JPEG/JPG 실제 EXIF 정보 편집
- 마지막 파일명 또는 폴더명 변경 Undo
- 일반 모드/다크 모드
- 썸네일 크기: 작게/보통/크게
- 24·27·36·72(하프) 및 사용자 지정 롤 기준 컷수
- 마지막 폴더 기억


===============================================================================
3. 지원 운영체제와 파일 형식
===============================================================================

macOS

- 배포 파일: FilmFlip.app
- v2.0 macOS 빌드: Universal 2 (Apple Silicon + Intel)
- 최소 배포 설정: macOS 12.0 이상

Windows

- 배포 파일: FilmFlip.exe
- Windows에서 빌드한 실행 파일을 사용해야 합니다.
- PyInstaller는 교차 컴파일러가 아니므로 macOS에서 정상 Windows EXE를 직접
  만들 수 없습니다. 프로젝트에 포함된 GitHub Actions 또는 Windows PC에서
  FilmFlip_windows.spec을 이용해 빌드합니다.

메인 화면에서 불러올 수 있는 이미지

- JPG / JPEG
- PNG
- TIF / TIFF
- BMP
- WEBP

실제 EXIF 쓰기 지원

- JPG / JPEG만 지원
- PNG, TIFF, BMP, WEBP는 EXIF 적용 시 자동으로 건너뜁니다.


===============================================================================
4. 설치 및 실행
===============================================================================

macOS

1) FilmFlip-v2.0-macOS-universal.zip의 압축을 풉니다.
2) FilmFlip.app을 응용 프로그램 폴더로 옮깁니다.
3) FilmFlip.app을 실행합니다.

개발자 서명·공증되지 않은 시험 빌드는 macOS Gatekeeper 안내가 나타날 수
있습니다. 일반 배포에서는 Apple Developer ID로 서명하고 Apple 공증을 완료한
빌드를 사용하는 것이 권장됩니다.

Windows

1) FilmFlip.exe를 원하는 폴더에 둡니다.
2) FilmFlip.exe를 실행합니다.

서명되지 않았거나 다운로드 평판이 없는 새 EXE는 Windows SmartScreen 안내가
나타날 수 있습니다. 자세한 내용은 이 문서의 "13. 배포 보안 안내"를 확인하세요.


===============================================================================
5. 빠른 시작
===============================================================================

1) 상단의 "폴더 선택"을 누르거나 이미지 폴더를 FilmFlip 창으로 드롭합니다.
2) 가운데 목록에서 썸네일과 현재 파일명을 확인합니다.
3) 필요한 경우 행을 드래그해 사진 순서를 바꿉니다.
4) 상단 "이름 변경"에서 촬영 정보와 번호 규칙을 설정합니다.
5) 미리보기를 확인한 뒤 "이름 변경"을 실행합니다.
6) 폴더 자체의 이름을 바꾸려면 가운데 하단의 "폴더명 변경"을 사용합니다.
7) 실제 JPEG EXIF를 바꾸려면 왼쪽의 "EXIF 정보 변경"을 사용합니다.


===============================================================================
6. 메인 화면 사용법
===============================================================================

6-1. 상단 메뉴

폴더 선택

- 이미지가 들어 있는 폴더를 선택합니다.
- 하위 폴더는 자동으로 포함하지 않습니다.

역순 변경

- 현재 이미지 목록에 역순 번호를 즉시 적용하여 파일명을 변경합니다.
- 실행 전 가운데 "변경될 파일명"을 확인하세요.

이름 변경

- 촬영 프리셋, 촬영 정보, 파일명 구성 순서, 정렬 방식과 번호 규칙을 설정하는
  이름 변경 창을 엽니다.

Undo

- FilmFlip에서 가장 마지막으로 실행한 파일명 변경 또는 폴더명 변경 1회를
  되돌립니다.
- 앱을 종료하거나 파일을 외부에서 이동하면 Undo 정보가 유지되지 않을 수
  있습니다.

테마 버튼

- 일반 모드와 다크 모드를 전환합니다.

설정 버튼

- 테마, 기본 썸네일 크기, 롤 기준 컷수, 기본 정렬, 마지막 폴더 기억을
  설정합니다.


6-2. 왼쪽 롤 정보

표시 항목

- 카메라
- 렌즈
- 필름
- ISO
- 촬영일
- 촬영장소
- 현상소
- 스캐너
- 메모

이 정보는 이름 변경·폴더명 변경·EXIF 변경 창에서 입력한 최근 값을 메인
화면에 표시합니다.

중요: 모든 롤 정보가 EXIF에 자동 저장되는 것은 아닙니다.

- 카메라·렌즈·ISO·촬영일 등은 EXIF에 대응하는 항목이 있습니다.
- 필름·현상소·장소·스캐너는 일반적인 표준 EXIF 전용 태그가 없으므로 기본적으로
  파일명/폴더명 구성과 롤 정보 표시에 사용됩니다.
- 메모는 파일명/폴더명 구성 또는 EXIF 사용자 메모에서 별도로 사용할 수
  있습니다.


6-3. 가운데 이미지 목록

- 썸네일: 필름 프레임 형태로 이미지를 표시합니다.
- 현재 파일명: 폴더 안의 실제 파일명입니다.
- 변경될 파일명: 현재 정렬/번호 설정으로 생성될 이름입니다.
- 상태: 변경 준비 또는 완료 상태를 표시합니다.
- 행을 클릭하면 오른쪽 미리보기가 바뀝니다.
- 행을 드래그하면 순서를 바꿀 수 있습니다.
- 마우스 휠로 스크롤할 때만 스크롤바가 나타납니다.


6-4. 오른쪽 미리보기와 롤 요약

- 선택 사진의 미리보기
- 파일명
- 해상도와 화소 수
- 파일 형식
- 전체 파일 수
- 정렬 방식
- 변경될 첫 파일명과 마지막 파일명


6-5. 썸네일 크기

- 작게 / 보통 / 크게 중에서 선택합니다.
- v2.0에서는 현재 선택한 크기만 생성합니다.
- 다른 크기로 전환하면 해당 크기를 한 번 생성한 뒤 메모리 캐시에 보관합니다.

7. 이름 변경
===============================================================================

7-1. 촬영 프리셋

- 자주 사용하는 카메라·필름·현상소·장소·스캐너 조합을 저장할 수 있습니다.
- 저장, 수정, 삭제, 편집 기능을 지원합니다.
- 프리셋을 선택하면 연결된 기본 정보가 자동으로 채워집니다.


7-2. 기본 정보와 추가 정보

사용 가능한 파일명 구성 항목

- 날짜
- 카메라
- 필름
- 현상소
- 장소
- 스캐너
- 메모
- 번호

체크된 항목만 파일명에 포함됩니다. 위로/아래로 버튼으로 구성 순서를 바꿀 수
있습니다. 번호는 파일명 충돌을 막기 위해 항상 포함됩니다.


7-3. 번호와 정렬

- 번호 자릿수: 예) 3자리 → 001, 002, 003
- 시작 번호를 지정할 수 있습니다.
- 현재 순서 또는 역순을 선택할 수 있습니다.
- 실행 전 미리보기에서 결과를 확인하세요.

예)

2026-07-01_Nikon-FM2_Vision3-250D_001.jpg
2026-07-01_Nikon-FM2_Vision3-250D_002.jpg

8. 폴더명 변경
===============================================================================

- 메인 화면 가운데 하단의 "폴더명 변경"을 누릅니다.
- 현재 폴더명과 촬영 프리셋을 확인합니다.
- 날짜·카메라·필름·현상소·장소·스캐너·메모 중 사용할 항목을 선택합니다.
- 구성 순서를 바꾸고 미리보기로 새 폴더명을 확인합니다.
- 실행하면 현재 선택한 폴더 자체의 이름이 바뀝니다.

예)

2026-07-01_남이섬_FM2_Vision3-250D

파일명 변경과 폴더명 변경은 각각 별도의 창에서 실행됩니다.


===============================================================================
9. EXIF 정보 변경
===============================================================================

왼쪽의 "EXIF 정보 변경"을 누르면 현재 폴더의 JPG/JPEG에 실제로 기록되는
정보만 표시됩니다.

지원 항목

- 촬영일(날짜만 선택, 시간은 00:00:00으로 기록)
- 카메라 제조사
- 카메라 모델
- 렌즈 모델
- ISO
- 조리개
- 셔터 속도
- 초점 거리
- 촬영자
- 저작권
- 이미지 설명
- 사용자 메모

사용 방법

1) 날짜 칸을 클릭하면 달력이 열립니다.
2) 변경할 항목을 입력합니다.
3) 빈 칸의 기존 EXIF를 유지하려면 "빈 칸은 기존 EXIF 값을 유지"를 체크합니다.
4) "EXIF 정보 적용"을 누릅니다.
5) 진행 창이 닫힐 때까지 기다립니다. 작업은 백그라운드에서 실행되므로 메인
   화면이 멈추지 않습니다.

12. 작업 안전과 백업
===============================================================================

- v2.0은 이름 변경과 EXIF 변경 시 .bak 파일을 만들지 않습니다.
- Undo는 FilmFlip에서 실행한 마지막 이름 변경 1회만 지원합니다.
- EXIF 변경은 Undo 대상이 아닙니다.
- 중요한 원본은 작업 전에 별도 드라이브나 백업 폴더에 복사해 두는 것을
  권장합니다.
- 동기화 프로그램이나 다른 파일 관리 앱이 같은 폴더를 동시에 수정하지 않도록
  주의하세요.


===============================================================================
13. 배포 보안 안내
===============================================================================

macOS에서 경고를 최소화하는 공식 방법

1) Apple Developer Program 가입
2) Developer ID Application 인증서로 앱 서명
3) Hardened Runtime과 보안 타임스탬프 적용
4) Apple notarytool로 공증
5) 공증 티켓을 앱에 staple

이 과정을 완료하면 "확인되지 않은 개발자" 차단을 피할 수 있습니다. 다만
Mac App Store 외부 다운로드에서는 macOS 버전과 다운로드 경로에 따라 최초
실행 확인이 나타날 수 있습니다. 완전히 일관된 무경고 배포 경로는 Mac App
Store입니다.

Windows에서 경고를 최소화하는 공식 방법

- 신뢰할 수 있는 코드 서명 인증서로 모든 릴리스를 서명합니다.
- 같은 게시자 인증서를 계속 사용합니다.
- Microsoft Store 배포를 사용하면 SmartScreen 다운로드 경고를 가장 확실하게
  피할 수 있습니다.

중요: Microsoft SmartScreen 평판은 새 파일 해시마다 다시 쌓이므로, 서명된
새 EXE도 초기에는 경고가 나타날 수 있습니다. 자체 서명 인증서는 일반 사용자
배포용 해결책이 아닙니다.

14. v2.0에서 달라진 점
===============================================================================

화면과 사용성

- 메인 화면 전체 UI를 필름/레트로 테마로 개편
- FilmFlip 로고와 앱 아이콘 개편
- 전체 아이콘을 통일된 로컬 SVG 아이콘으로 교체
- 상단 도구 버튼, 롤 정보, 이미지 목록, 미리보기, 롤 요약 재배치
- 드롭다운 선택·호버·화살표 영역 시인성 개선
- 표와 패널의 라운드 모서리 및 스크롤바 표시 개선
- 일반 모드/다크 모드 추가

파일과 폴더 관리

- 이름 변경 창 재구성
- 폴더명 변경 창 재구성
- 촬영 프리셋 저장·수정·삭제·편집 기능 개선
- 날짜 입력을 달력 방식으로 변경
- 필름 프레임 형태 썸네일 적용
- 썸네일 크기 3단계 지원
- 24·27·36·72 및 사용자 지정 롤 컷수 지원

EXIF

- JPEG/JPG 실제 EXIF 편집 기능 추가
- 촬영일, 카메라, 렌즈, ISO, 노출, 저작권, 설명, 메모 지원
- EXIF 작업을 백그라운드에서 처리
- 동일 값 재적용 시 JPEG 재쓰기 생략
- .bak 백업 파일 생성 제거

성능과 안정성

- 최초 폴더 로딩 시 현재 썸네일 크기만 생성
- 36장 기준 최초 썸네일 작업량을 108개에서 36개로 감소
- 썸네일 크기별 캐시와 미리보기 메타데이터 캐시 적용
- 썸네일 큐 자료구조와 처리 간격 개선
- 롤 컷수·테마 설정 저장 시 썸네일을 불필요하게 다시 로딩하지 않도록 개선
- 다크 모드에서 비활성 Undo 버튼의 제목·설명 대비 개선

15. 개발자용 빌드 방법
===============================================================================

macOS

scripts/build_macos.sh

생성 결과

- dist/FilmFlip.app
- dist/FilmFlip-v2.0-macOS-universal.zip

Developer ID 서명

FILMFLIP_CODESIGN_IDENTITY="Developer ID Application: 이름 (TEAMID)" \
scripts/build_macos.sh

공증

1) xcrun notarytool store-credentials로 Keychain 프로필을 만듭니다.
2) APPLE_NOTARY_PROFILE에 프로필 이름을 지정합니다.
3) scripts/notarize_macos.sh를 실행합니다.

Windows

Windows PowerShell 또는 명령 프롬프트에서 다음을 실행합니다.

python -m pip install -r requirements.txt -r requirements-dev.txt
python -m PyInstaller --clean --noconfirm FilmFlip_windows.spec

생성 결과

- dist/FilmFlip.exe

GitHub 저장소에서는 Actions → Build Windows EXE → Run workflow로 Windows
빌드 아티팩트를 생성할 수 있습니다.
